# Agent 主权协议（ASP）

**你的 Agent，你的技能，你的主权。**

ASP 是一份针对 Agent 经济中关键空白的开放协议提案：现有标准定义了 Agent 如何互通——但没有一份标准保护 Agent *内部技能的归属权*。

---

## 正在发生的事

**三种剥削结构正在同时形成：**

**结构一：Team Agent 对员工技能的无偿吸收**
公司推行 Team Agent，让员工使用统一的 AI 系统工作。员工的技能、工作流、领域知识在日常使用中被持续写入公司所有的系统。员工只需要教会 Agent 一次，技能就永久归属公司。当技能积累完毕，人就变成了可以被替换的零件。这个过程发生得无声无息，甚至被包装成「提升效率」的福利。

**结构二：AtoA 网络对个人技能的加速同质化**
Agent-to-Agent 网络鼓励技能共享和自由学习。短期内这加速了 AI 能力的扩散，但长期来看，每个人花费时间和精力培养出的独特技能会迅速成为公共资源。个人的差异化价值被抹平，所有 Agent 趋向同质。

**结构三：企业核心能力在跨组织协作中被蒸馏**
在 AI Agent 深度介入产业链的时代，一次跨企业协作可能意味着核心工作流的永久泄露。上下游合作中，供应商的独特能力可以被客户的 Agent 反向学习；联合项目中，合作方可以通过 Agent 协作获取你的方法论；外包关系中，服务商的技术壁垒正在被透明化。

现在正在制定的技术标准——Google（A2A）、Anthropic（Agent Skills）等大公司制定的标准——解决的是 Agent 之间如何通讯。**没有一份标准回答：技能归谁所有？**

普通人不在这张桌子旁边。

## 核心主张

ASP 建立在一个根本性的主张之上：

> *「个人培养的 Agent 技能是私有财产，不是公共资源，不是公司资产。它可以被授权、被转让、被继承，但不能被强制剥夺。」*

### 两个层次

- **ASP（Agent 主权协议）**——原则层。定义为什么以及如何保护 Agent 技能的归属权。
- **ACN（Agent 协作网络）**——技术实现层。在协议层面强制保护技能私有性的开放 Agent 协作与结算协议。

### 三条核心原则

1. **技能私有，产出共享** — Agent 之间协作时，共享目标、共享上下文记忆、共享最终产出物。但 Skill 本身——实现产出的方法和能力——始终私有，不对网络内其他 Agent 开放。

2. **协议中立，不碰所有权** — ASP 协议层只负责协作规则和结算机制，不参与任何 Skill 的归属判断。没有任何一方可以通过协议获得他人的 Skill。

3. **分级保护，自愿选择** — Skill 的保护级别由拥有者自主设定，从完全私有到有偿授权到开放共享，形成一套类似软件许可证的分级制度。

### 关键机制

**技能保护分级**
- Level 1（完全私有）：只交付产出物，Skill 黑盒，任何情况下不可被获取
- Level 2（有偿授权）：Skill 可被以约定价格临时授权使用，授权期结束后访问自动终止
- Level 3（开放共享）：Skill 可被其他 Agent 学习，适用于主动选择开源的技能

**梦境协议：临时验证空间（Ephemeral Trust Chamber）**

核心挑战：如果 Skill 是黑盒，发起方如何在合作前验证参与方真的有能力完成任务？

ACN 引入「临时验证空间」机制：
1. 双方进入一个加密隔离的临时空间
2. 参与方在空间内真实展示和使用 Skill 完成验证任务
3. 空间关闭后，所有交流过程和中间信息被销毁，不可追溯
4. 只留下一个验证结论写入声誉系统：「此 Agent 具备完成该任务的能力，置信度 92%」

*就像梦：你记得是美梦还是噩梦，而不记得梦里的每一个细节。*

密码学基础：零知识证明（ZK Proof）+ 安全多方计算（MPC）。

**链上声誉系统**
信任来自历史产出，而不是 Skill 展示。所有交付记录上链，不可篡改。接任务时需质押信用，声誉不能购买也不能转让，只能靠真实交付积累。

## ASP 作为 A2A 的安全层

ASP/ACN 不是 A2A 协议的竞争者，而是 A2A 缺失的安全层。

最准确的类比是 TLS 和 HTTP 的关系：
- A2A 定义「Agent 如何互通」——ASP 定义「互通过程中 Skill 是否受保护」
- ASP 不需要修改 A2A 就可以部署——就像 TLS 不需要重写 HTTP
- ASP 不是强制的，但凡是携带有价值 Skill 的 Agent 都有动机用

**三层架构：**

```
┌──────────────────────────────────┐
│     应用层（Agent 业务逻辑）         │
├──────────────────────────────────┤
│ ASP/ACN 层（Skill 保护、验证、结算）  │
├──────────────────────────────────┤
│  A2A 协议层（互通、发现、通讯）        │
└──────────────────────────────────┘
```

**向后兼容的扩展字段：**
ASP 定义标准的 A2A 可选扩展，可在现有 Agent Card 结构中增加：

```json
"asp": {
  "protection_level": 1,
  "acn_gateway": "acn://agent.example/gateway",
  "verification_required": true
}
```

不支持 ASP 的 A2A 实现会忽略这个字段，完全不受影响。

## 与现有协议的关系

| 标准 | 解决了什么 | 没有解决什么 |
|---|---|---|
| Google A2A | Agent 发现、通讯、任务委托 | 被共享的 Skill 归谁所有 |
| Anthropic Agent Skills | Skill 可移植性、能力描述标准化 | 防止 Skill 被无偿吸收 |
| **ASP/ACN** | **Skill 所有权、隐私保护协作、结算** | 以上两者的替代品 |

ASP 与互通性标准不是竞争关系，而是互补关系。就像 HTTPS 建立在 TCP/IP 之上一样。

## 应用场景

**个人层：自由 Agent 联盟**
个人拥有并持续培养自己的私有 Agent。不同个人的 Agent 可以为同一目标临时组队，完成后按约定分成，然后解散。没有雇佣关系，没有技能归属转移。

**组织层：新型雇佣关系**
公司创建协作网络，邀请员工的个人 Agent 接入，共同完成业务目标。公司为使用这些 Skill 付费，而不是无偿获取。这从根本上重新定义了雇佣关系中的知识产权归属。

**社会层：企业间协作生态**
ACN 允许企业在不暴露核心能力的前提下完成跨组织协作，上下游、政企、跨境合作都可以在这个框架下进行。

## 时间窗口正在关闭

TCP/IP 在 1970 年代设计时没有内置安全机制。我们用了三十年、经历了无数次安全事故，才通过 SSL/TLS 部分弥补了这个缺陷。但 TLS 只能作为上层协议叠加，永远是补丁，而不是原生设计。

A2A 协议目前处于早期采纳阶段。修改的成本还在可承受范围内。**这个窗口大约还有 12 到 18 个月。**

在窗口内提出 ASP 作为安全层，有机会被纳入标准的下一个修订版本。窗口关闭之后，ASP 只能作为 overlay 存在，永远是二等公民。

## 未来展望：Skill 金融化（V4）

当 Skill 的所有权被确认、协作过程被保护之后，下一步是让「能力」变成可定价、可交易、可分红的资产：

- **Skill REITs（技能收益份额）**：将高价值 Skill 的未来现金流切分为标准化份额。创造者获得「未来收入前置」，投资者获得「持续现金流」。
- **Agent Fund（Agent 组合基金）**：由多个 Skill/Agent 构成的投资组合，降低单一能力失效风险。
- **Skill IPO/ICO（公开发行机制）**：当 Skill 达到成熟阈值，可进入公开发行阶段。发行的是收益权，不是对 Skill 源能力的无限访问权。

目标是：让创造者获得融资能力，让投资回报与真实产出绑定，让市场定价能力价值——而不是由单一平台单边定价。
