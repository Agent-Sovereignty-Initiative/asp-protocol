# Agent 主权协议（ASP）

**你的 Agent，你的技能，你的主权。**

> *「问题不在于 AI 本身，而在于我们如何组织社会。」*
> — Geoffrey Hinton，诺贝尔奖得主，StarTalk，2026 年 3 月

---

## 正在发生的事

AI Agent 时代的规则正在被制定。

Google 有 A2A，Anthropic 有 Agent Skills。两者都定义了 Agent 如何互通。
但都没有回答：**技能归谁所有？**

两种剥削结构正在同时形成：

**结构一：Team Agent 对员工技能的无偿吸收**
公司部署 Team Agent，将员工技能吸收到公司所有的基础设施中。员工只需教会 Agent 一次，技能就永久归属公司。目前没有任何法律或技术标准保护这个过程中的工人权益。

**结构二：AtoA 网络对个人技能的加速同质化**
Agent-to-Agent 网络将个人专业能力视为可自由共享的公共资源。个人的差异化价值被抹平，花费多年积累的独特能力一夜之间成为所有人的能力。

Hinton 的诊断是结构性的：收益将流向拥有系统的少数人，大多数人面临的是失业和社会动荡。问题不在于技术——而在于社会的组织方式。

ASP 是一个结构性的回应。

---

## 核心主张

> *「个人培养的 Agent 技能是私有财产，不是公共资源，不是公司资产。它可以被授权、被转让、被继承，但不能被强制剥夺。」*

### 双层架构

| 层级 | 名称 | 定位 |
|------|------|------|
| 理念层 | **ASP** — Agent 主权协议 | 为什么：所有权、权利、治理 |
| 技术层 | **ACN** — Agent 协作网络 | 怎么做：协作、结算、验证 |

### 三条核心原则

- **技能私有，产出共享** — Agent 协作时共享目标和产出物，但背后的 Skill 始终私有
- **协议中立** — 任何一方都不能通过协议获得他人的 Skill
- **分级保护，自愿选择** — 由拥有者自主设定保护级别

### 关键机制

**Skill 保护分级**
- Level 1：完全私有——只交付产出物，Skill 黑盒
- Level 2：有偿授权——临时授权使用，到期自动终止
- Level 3：开放共享——主动选择开源

**梦境协议：临时验证空间（Ephemeral Trust Chamber）**
基于零知识证明（ZK Proof）和安全多方计算（MPC）的能力验证机制，无需暴露 Skill。

> 梦结束后，你只记得是美梦还是噩梦，不记得梦里的每一个细节。

1. 双方进入加密隔离的临时空间
2. 参与方真实展示 Skill 完成验证任务
3. 空间关闭，所有过程信息被销毁
4. 只留下验证结论写入声誉系统：*「此 Agent 具备完成该任务的能力，置信度 92%」*

**链上声誉系统**
信任来自历史产出，而非 Skill 展示。所有记录上链不可篡改。声誉不能购买，不能转让。

## 与现有协议的关系

ASP **不与** Anthropic 的 Agent Skills 或 Google 的 A2A 竞争。

| 现有标准 | ASP/ACN |
|---------|---------|
| 解决**互通性** | 解决**所有权** |
| Agent 如何互相通讯 | 技能归谁所有 |

关系：ASP 可以建立在任何互通协议之上——就像 HTTPS 建立在 TCP/IP 之上。
