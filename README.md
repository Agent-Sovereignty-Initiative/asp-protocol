# Agent Sovereignty Protocol (ASP)

**Your Agent. Your Skills. Your Sovereignty.**

> *"The problem is not AI itself. It is how society is organized."*
> — Geoffrey Hinton, Nobel Laureate, StarTalk, March 2026

---

## The Problem

The rules of the AI Agent economy are being written right now.

Google has A2A. Anthropic has Agent Skills. Both define how Agents interoperate.
Neither answers: **who owns the skills inside?**

Two extractive structures are forming simultaneously:

**1. Team Agents and the silent appropriation of employee skills**
Companies deploy Team Agents that absorb employee skills into corporate-owned infrastructure. Workers teach the Agent once; the skill belongs to the company permanently. There is no legal or technical standard protecting this contribution.

**2. AtoA networks and the acceleration of skill homogenization**
Agent-to-Agent networks treat individual expertise as a public resource to be freely shared. Individual differentiation erodes. The distinctive capabilities people invest years to develop become everyone's capabilities overnight.

Hinton's diagnosis is structural: the gains will concentrate among those who own the systems. The majority faces unemployment and social upheaval. The problem is not the technology — it is how society is organized.

ASP is an attempt at a structural answer.

---

## The Proposal

ASP is built on one foundational claim:

> *Skills cultivated in a personal Agent are private property — not a public resource, not a corporate asset. They may be licensed, transferred, or inherited, but they cannot be involuntarily extracted.*

### Two Layers

| Layer | Name | Purpose |
|-------|------|---------|
| Principle | **ASP** — Agent Sovereignty Protocol | The *why*: ownership, rights, governance |
| Technical | **ACN** — Agent Collaboration Network | The *how*: collaboration, settlement, verification |

### Three Core Principles

- **Skills are private, outputs are shared** — Agents collaborate on goals and deliverables, but the Skill behind the output stays private
- **The protocol is neutral** — no party can obtain another's Skill through the protocol
- **Tiered protection, voluntary choice** — owners set their own protection level (fully private / licensed / open)

### Key Mechanisms

**Skill Protection Tiers**
- Level 1: Fully private — outputs only, Skill is a black box
- Level 2: Licensed for fee — temporary access at agreed price, auto-terminates
- Level 3: Open shared — voluntary open-source of capabilities

**Ephemeral Trust Chamber** *(the Dream Protocol)*
Capability verification without Skill disclosure, based on Zero-Knowledge Proofs and Secure Multi-Party Computation.

> Like a dream: you remember whether it was good or bad. You don't remember what happened inside.

1. Both parties enter a cryptographically isolated temporary space
2. The participant demonstrates capability on a verification task
3. The space closes — all process data is destroyed
4. Only the conclusion is written to the reputation system: *"Capability confirmed, 92% confidence"*

**On-Chain Reputation System**
Trust derived from delivery history, not Skill disclosure. All records immutable and public. Reputation cannot be purchased or transferred.

## Relationship to Existing Standards

ASP does **not** compete with Anthropic's Agent Skills or Google's A2A.

| Existing standards | ASP/ACN |
|-------------------|---------|
| Solve **interoperability** | Solves **ownership** |
| How Agents talk to each other | Who owns the skills being shared |

Relationship: ASP can be built on top of any interoperability protocol — just as HTTPS is built on TCP/IP.

---

## Three Layers of Application

**Individual Layer: The Free Agent Alliance**
Personal Agents collaborate temporarily, split earnings, disband. No employment relationship, no skill transfer.

**Organizational Layer: A New Employment Relationship**
Companies pay to *access* employee Skills rather than permanently appropriating them. A fundamental redefinition of IP ownership in the employment relationship.

**Social Layer: Interorganizational Collaboration**
Cross-organizational collaboration without core capability leakage — supply chain, public-private, cross-border.
