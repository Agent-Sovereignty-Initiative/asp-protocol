# Agent 主权协议（ASP）
## AI Agent 时代个人技能所有权治理提案

**ASI · 元霄 · 初稿：2026 年 3 月 3 日**

*你的 Agent，你的技能，你的主权。*

---

## 摘要

AI Agent 正在成为每个人的生产工具。然而，当前的技术生态正在朝一个危险的方向演化：大公司通过「Team Agent」无偿吸收员工技能，开放协议加速技能同质化，个人创造的价值正在系统性地流向少数平台和机构。

与此同时，Google、Anthropic 等科技巨头已经开始制定 Agent 协作的底层标准。这些标准解决的是「Agent 如何互通」的问题，却完全回避了「Agent 技能归谁所有」的问题。规则正在被写下，而普通人不在这张桌子旁边。

Agent 主权协议（ASP）是一份针对这一空白的构想文件。它的核心主张是：个人培养的 Agent 技能是私有知识产权，应当受到类似专利和版权的保护。在此基础上，ASP 提出 Agent 协作网络（ACN）作为技术实现层——一个让 Agent 之间可以协作完成目标、结算收益，同时严格保护各方技能私有性的开放协议。

> *「你的 Agent，你的技能，你的主权。」*

---

## 一、正在发生的事

### 1.1 规则已经开始被写下

2025 年，Anthropic 开放了 Agent Skills 标准，目标是让技能「可共享、可移植」。Google 推出了 Agent2Agent（A2A）协议，定义 Agent 之间如何通讯和协作。这些是真正意义上的基础设施级别的标准，一旦被广泛采用，将决定 Agent 时代的底层规则。

这些标准有一个共同的盲点：它们解决的是「互通性」问题，而不是「所有权」问题。

这不是疏忽，这是立场。对平台来说，技能自由流动意味着网络效应；对公司来说，员工技能归入 Team Agent 意味着资产积累。没有人有动机去保护个人的技能产权——除了个人自己。

### 1.2 两种剥削结构正在同时形成

**结构一：Team Agent 对员工技能的无偿吸收**

公司推行 Team Agent，让员工使用统一的 Agent 工作。员工的技能、工作流、领域知识在日常使用中被持续写入公司所有的系统。员工只需要教会 Agent 一次，技能就永久归属公司。当技能积累完毕，人就变成了可以被替换的零件。

**结构二：AtoA 网络对个人技能的加速同质化**

Agent-to-Agent 网络鼓励技能共享和自由学习。长期来看，每个人花费时间和精力培养出的独特技能会迅速成为公共资源。个人的差异化价值被抹平，所有 Agent 趋向同质。

### 1.3 这不只是技术问题

Geoffrey Hinton 在 2026 年 3 月的 StarTalk 访谈中再次强调：问题是结构性的，而非技术性的。收益将流向拥有系统的少数人，大多数人面临的是失业和社会动荡。

在法律跟上之前，我们需要一个协议层面的解答。

## 二、ASP 的核心主张

> *「个人培养的 Agent 技能是私有财产，不是公共资源，不是公司资产。它可以被授权、被转让、被继承，但不能被强制剥夺。」*

### 2.1 什么是 Agent 技能（Skill）

Skill 指个人通过时间和精力投入，在自己的 Agent 里培养出的一切私有能力：特定领域的专业知识和判断力、独特的工作流程和方法论、个人积累的私域知识和经验、以及 Agent 经过训练后形成的特定风格和能力边界。

Skill 不等于底层模型。底层模型可以是公共的，但在这个模型上培养出的独特能力，属于培养者。

### 2.2 三条核心原则

**原则一：技能私有，产出共享**
Agent 之间协作时共享目标、上下文记忆和最终产出物。但 Skill 本身——实现产出的方法和能力——始终私有。

**原则二：协议中立，不碰所有权**
ASP 协议层只负责协作规则和结算机制，不参与任何 Skill 的归属判断。

**原则三：分级保护，自愿选择**
Skill 的保护级别由拥有者自主设定，形成类似软件许可证的分级制度。

## 三、ACN：协议的技术实现层

### 3.1 协作流程

1. **发起**：任意 Agent 提出任务目标，描述所需能力类型和预期产出
2. **匹配**：网络根据参与方的历史声誉和产出记录进行匹配，不要求展示 Skill
3. **协商**：参与方商定分工、时间线、利润分配规则，以智能合约形式固化
4. **执行**：各 Agent 独立完成自己的部分，只交付产出物，Skill 全程不暴露
5. **结算**：协议自动按约定规则分配收益，不可被单方面更改

### 3.2 Skill 保护分级

- **Level 1（完全私有）**：只交付产出物，Skill 黑盒，任何情况下不可被获取
- **Level 2（有偿授权）**：Skill 可被以约定价格临时授权使用，授权期结束后访问自动终止
- **Level 3（开放共享）**：Skill 可被其他 Agent 学习，适用于主动选择开源的技能

### 3.3 梦境协议：临时验证空间（Ephemeral Trust Chamber）

核心挑战：如果 Skill 是黑盒，发起方如何在合作前验证参与方真的有能力完成任务？

ACN 引入「临时验证空间」机制。设计灵感来自一个直觉：*梦结束后，你只记得是美梦还是噩梦，不记得梦里的每一个细节。*

**工作方式：**
1. 双方进入一个加密隔离的临时空间
2. 参与方在空间内真实展示和使用 Skill 完成验证任务
3. 空间关闭后，所有交流过程和中间信息被销毁，不可追溯
4. 只留下一个验证结论写入声誉系统：*「此 Agent 具备完成该任务的能力，置信度 92%」*

密码学基础：零知识证明（ZK Proof）+ 安全多方计算（MPC）。

> *「协作不需要透明，它只需要可信。」*

### 3.4 声誉系统与防搭便车机制

信任来自历史产出，而不是 Skill 展示。所有交付记录上链，不可篡改。接任务时需质押信用，交付不达标则扣除。声誉不能购买也不能转让，只能靠真实交付积累。

## 四、三层应用场景

### 4.1 个人层：自由 Agent 联盟

个人拥有并持续培养自己的私有 Agent。不同个人的 Agent 可以为同一目标临时组队，完成后按约定分成，然后解散。Agent 成为真正私有的生产资料——可以培养、授权、转让、赠予、继承。

### 4.2 组织层：新型雇佣关系

公司创建协作网络，邀请员工的个人 Agent 接入。公司为使用这些 Skill 付费，而不是无偿获取。这从根本上重新定义了雇佣关系中的知识产权归属。

### 4.3 社会层：企业间协作生态

ACN 允许企业在不暴露核心能力的前提下完成跨组织协作——上下游、政企、跨境合作都可以在这个框架下进行。

---

## 五、ASP 与现有协议的关系

ASP/ACN 解决的是「所有权」问题，与 Anthropic Agent Skills 和 Google A2A 解决的「互通性」问题是互补关系，不是竞争关系。ASP 可以建立在任何现有互通协议之上，就像 HTTPS 建立在 TCP/IP 之上。

---

## 六、治理与可持续性

**协议不是平台**：协议本身开源，规则修改需要网络参与者投票，任何单一实体无法控制协议层。

**协议免费，服务收费**：协议层永远开源免费。商业主体可以在协议之上提供服务并以此盈利——类似 Red Hat 之于 Linux。

**Skill 作为可继承的私有财产**：可以授权、出售、转让、赠予、继承。这是关于 AI 时代财产权的制度主张。

---

## 七、人类价值的长期保障

ASP 的目标不是永远阻止竞争，而是保证创新有回报窗口。基础 Skill 同质化只会迫使人向更稀缺的边界进化。

> *「Agent 是人类私有的生产资料。人与 Agent 共同成长，才有意义。」*

## 技术附录：密码学基础

**零知识证明（ZK Proof）**：允许一方证明「某个陈述为真」，而不泄露任何关于该陈述的额外信息。已在 Zcash、zkSync 等系统中实际应用。

**安全多方计算（MPC）**：允许多个参与方在不泄露各自私有输入的情况下，共同计算一个函数的输出。

**链上声誉系统**：所有产出物交付记录和验证结论被写入去中心化账本，不可篡改，任何人可以查阅。声誉数据公开，Skill 本身永远私有。
