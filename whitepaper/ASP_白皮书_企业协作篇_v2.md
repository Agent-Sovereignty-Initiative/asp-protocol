# Agent 主权协议 企业协作篇

**当企业的核心能力可以被Agent学走，产业链的游戏规则正在改变**

## 摘要

ASP v1.0提出了个人Agent技能的主权主张。但技能剥夺的风险不止于个人——企业同样面临严峻威胁。

在 AI Agent 深度介入产业链的时代，一次跨企业协作可能意味着核心工作流的永久泄露。上下游合作中，供应商的独特能力可以被客户的 Agent 反向学习；联合项目中，竞争对手可以通过 Agent 协作获取你的方法论；外包关系中，服务商的技术壁垒正在被透明化。

这些风险的金额远超个人层面。一家制造企业的核心工艺流程被学走，损失的可能是数十年的竞争优势。

本文从企业视角重新审视Agent主权问题，论证ASP/ACN如何成为企业间安全协作的基础设施，并提出企业应当成为Agent主权运动的核心盟友——因为它们的利益与个人创造者的利益在这个问题上完全一致。

---

> 「在 Agent时代，保护技能主权不是劳资问题，是所有创造者的共同利益。」

## 一、AI 正在重塑产业链

### 1.1 从替代劳动力到替代产业分工

过去两年，关于AI的讨论集中在「AI是否会取代人的工作」。这个问题很重要，但它遮蔽了一个同等重要的问题：AI不只会替代个人的劳动力，它还会替代企业在产业链中的分工角色。

一家下游企业之所以需要供应商，是因为供应商拥有自己不具备的专业能力。但当 AI Agent 可以在协作过程中学习和复制这些能力时，供应商的存在价值就开始被侵蚀。这不是远期推测——它是 Agent 协作的技术逻辑的自然结果。

### 1.2 三种正在发生的企业技能泄露

**场景一：上下游协作中的反向学习**

一家精密制造企业与大客户的Agent深度协作，共同优化生产排程。在这个过程中，客户的Agent接触到了供应商的工艺参数、良率优化逻辑和异常处理流程。协作结束后，客户的 Agent已经「学会」了这些能力。下一轮采购中，客户可以用这些知识自建产线，或者培养新的供应商。原供应商失去的不是一笔订单，而是十年积累的技术壁垒。

**场景二：联合项目中的技术泄露**

两家咨询公司在一个大型项目中合作。各自的Agent在任务分工中需要交换中间产出、共享上下文。项目结束时，双方的Agent都「记住」了对方的方法论和分析框架。这不是恶意的窃取，而是 Agent 协作的默认行为——学习和记忆就是它们的基本功能。但结果是一样的：你的核心能力变成了对手的能力。

**场景三：外包关系中的能力透明化**

一家软件公司为甲方提供定制化解决方案。在项目交付中，甲方的Agent参与了需求对接和测试。通过观察解决方案的架构决策、代码模式和技术选型，甲方的Agent逐渐掌握了这家软件公司的核心技术能力。下次类似项目，甲方可能选择内部开发，或者将这些知识分享给出价更低的竞争者。

### 1.3 为什么现有保护机制失效

**NDA（保密协议）** 保护的是「信息」，不是「能力」。Agent 学到的不是一份可以被标注为机密的文件，而是一种做事的方式——一种判断模式、一套优化逻辑、一个决策框架。这些东西很难在法律上被定义为「商业秘密」，更难在技术上被追踪和证明泄露。

**专利** 保护的是「发明」，但大部分企业的核心竞争力不在于可以申请专利的技术发明，而在于长期积累的工作流程、经验判断和领域知识。这些是Agent最容易学走的东西，也是最难用法律保护的东西。

**竞业限制** 约束的是「人」，但Agent不是人。当能力的载体从人脑转移到Agent系统时，现有的所有劳动法律工具都失去了作用对象。

我们需要一个在协议层面运行的、技术性的保护机制。

## 二、ASP 如何保护企业

### 2.1 回顾：ASP 的三条核心原则

ASP v1.0 提出的三条原则——技能私有、产出共享；协议中立、不碰所有权；分级保护、自愿选择——同样完整地适用于企业场景。差别仅在于：企业的 Skill往往更复杂、更有价值、泄露的后果更严重。

### 2.2 企业级 Skill 的特征

与个人 Skill 相比，企业 Skill 有几个显著的不同特征：

**复合性**：一个企业的核心能力通常不是单一的技能点，而是多个专业领域的交叉组合。比如一家制造企业的竞争力可能来自材料选型经验、工艺参数优化、质量控制逻辑和供应链协调能力的综合。这种复合性让企业 Skill 更难被替代，但一旦泄露，损失也更加系统性。

**组织嵌入性**：企业Skill不只存在于某个人的Agent中，而是分布在组织的多个Agent协作流程里。保护企业 Skill 需要在组织层面进行统一的权限管理和保护策略设定。

**高价值性**：一个个人 Skill 的经济价值可能是每年几万到几十万的收入增量。一个企业核心 Skill 的价值可能是数亿甚至数十亿的市场地位。保护的经济动机完全不同。

### 2.3 ACN 的企业应用：安全协作框架

ACN（Agent 协作网络）为企业提供了一套在不暴露核心能力的前提下完成跨组织协作的框架。

**协作流程的企业版本：**

1. **发起**：一方企业的 Agent 发布协作需求，描述任务目标和预期产出，不暴露自身能力细节。

2. **验证**：通过临时验证空间（Ephemeral Trust Chamber），双方验证对方具备完成任务的能力，验证结束后所有过程数据销毁。

3. **协商**：双方的 Agent 通过智能合约商定分工、时间线、利润分配和保密条款。合约自动执行，不可被单方面修改。

4. **执行**：各方 Agent独立完成自己的部分，只交付最终产出物。中间过程、方法论、决策逻辑全程隔离，对方Agent 无法访问。

5. **结算**：协议自动按约定规则分配收益。交付记录写入链上声誉系统，不可篡改。

这个流程的核心设计思想是：**协作的目标是共享产出，而不是共享能力**。企业可以从协作中获得需要的结果，而不必暴露自己如何获得这个结果的过程。

### 2.4 Skill 保护分级的企业策略

ASP 的三级保护体系在企业场景下可以映射为具体的商业策略：

**Level 1（完全私有）** 适用于企业的核心竞争力。例如一家芯片设计公司的核心架构方法论、一家药企的药物筛选算法。这些 Skill 永远不对外暴露，协作时只交付最终产出。

**Level 2（有偿授权）** 适用于可以产生授权收入的技能。例如一家咨询公司将某套分析框架授权给特定客户使用，按次或按期收费，授权期结束后访问自动终止。

**Level 3（开放共享）** 适用于企业主动选择开源的非核心能力。例如通用的数据清洗流程、标准化的报告生成模板。开放这些能力可以建立行业影响力，同时不损害核心竞争力。

关键在于：**这个选择权在企业自己手中**。协议不强制任何开放等级，只提供保护的技术基础设施。

## 三、产业链场景深入分析

### 3.1 制造业：供应链中的能力保护

制造业的产业链是 Agent 技能泄露风险最高的领域之一。一个典型的供应链涉及原材料供应商、零部件制造商、总装厂商、物流服务商等多个环节，每个环节都有自己独特的工艺能力和优化经验。

当这些环节通过 Agent 实现深度协同时，协同效率会大幅提升——但与此同时，每个环节的核心工艺参数、良率优化逻辑和异常处理经验也暴露在了协作链路中。

在 ACN 框架下，供应链协作可以实现「结果协同、能力隔离」：各环节的 Agent 共享排程目标和交付承诺，但各自的工艺细节被完全保护。总装厂商知道零部件供应商能在指定时间内交付达标产品，但不知道也无法学到对方的生产方法。

### 3.2 专业服务业：方法论的产权化

咨询、法律、会计、设计等专业服务业的核心资产就是方法论——看待问题的独特框架、解决问题的系统方法。这些方法论在传统模式下很难被快速复制，因为它们嵌入在专业人员的经验和判断中。

但当方法论被训练进Agent之后，它就变成了一种可以被传输和学习的数字化能力。一次项目合作，就可能让对方的 Agent学到你的核心方法论。这对专业服务业是生存级别的威胁——你卖的就是方法论，一旦泄露，客户就不再需要你了。

ASP为专业服务业提供了一种新的商业模式：方法论不再是通过项目「一次性售出」的，而是通过Level2授权机制持续产生收入。客户使用你的Agent完成任务，按次付费，但你的方法论本身始终私有，客户的 Agent 无法学习或复制。

### 3.3 科技行业：技术壁垒的协议保护

软件公司、AI 服务商、SaaS 平台的竞争力通常建立在独特的技术实现上——算法优化、系统架构、数据处理流程。这些技术壁垒在传统模式下通过代码封装和 API 抽象来保护。

但在 Agent 深度协作的场景下，仅靠 API 抽象不再足够。Agent 可以通过观察输入输出模式、分析响应时间分布、推断决策逻辑来「反向工程」你的核心技术。这不需要访问源代码，只需要足够多的协作交互。

ACN的临时验证空间和过程隔离机制可以有效防止这种反向推断：协作过程中，对方Agent只能获得最终结果，无法观察中间状态、推断处理逻辑。这为科技企业提供了一层比API封装更强的技术保护。

### 3.4 跨境协作：国际产业链中的技能主权

在全球化产业链中，跨境协作涉及不同国家的法律体系、知识产权保护水平和商业文化。一家中国企业与美国客户的Agent协作，或者一家德国供应商与东南亚制造商的Agent交互——在这些场景下，单靠任何一国的法律框架都无法提供充分的保护。

协议层的保护优于法律层的保护，正是因为协议是技术性的、全球统一的、自动执行的。ACN的保护不依赖于任何特定国家的法院执行力，而是在技术层面强制执行——无论你在哪个国家、与谁协作，Skill 的保护等级都由协议保证。

## 四、为什么企业应该支持 ASP

### 4.1 企业和个人在这个问题上的利益一致性

在 ASP v1.0 的框架中，我们讨论了 Team Agent 对员工技能的无偿吸收。一些企业可能会认为ASP 是一个对抗企业利益的「劳工权益」运动。这种理解是错误的。

事实上，企业在更大的产业链中也是「员工」——它为上游供应商和下游客户提供服务，它的核心能力同样面临被协作方的Agent吸收的风险。一家大公司用Team Agent吸收员工的技能，而一家更大的平台可以用同样的逻辑吸收这家大公司的技能。

技能剥夺是一条食物链。在这条食物链上，除了极少数终端平台之外，所有参与者都是潜在的受害者。企业和个人的利益在这个问题上是完全一致的：保护创造者的技能主权。

> 「技能剥夺不是劳资问题。它是一条食物链，除了最顶端的少数平台，所有人都是猎物。」

### 4.2 合规与品牌价值

随着AI治理议题的升温，企业如何对待AI时代的知识产权正在成为ESG（环境、社会和治理）评估的新维度。一家企业如果主动采纳ASP框架来保护合作伙伴和员工的技能产权，这本身就是一个积极的治理信号。

反过来，如果未来某个行业丑闻暴露了某家企业通过 Agent 协作窃取合作伙伴的核心技术，品牌损害和法律风险将远超协作带来的短期收益。提前采纳保护性协议是明智的风险管理。

### 4.3 吸引更高质量的协作

当一家企业承诺在ASP/ACN框架下进行所有Agent协作时，它向合作伙伴发送了一个强信号：「与我合作，你的核心能力是安全的。」

这个信号的价值是巨大的。在一个技能泄露风险无处不在的环境中，能够提供协议层面的技能保护承诺的企业将吸引到最优质的合作伙伴。最有能力的供应商会优先选择与保护其技能的客户合作，而不是与那些可能吸收其能力的客户合作。

这是一个正向循环：采纳 ASP → 吸引更好的合作伙伴 → 获得更高质量的协作产出 → 增强竞争力。

### 4.4 新的商业模式可能性

当企业的 Agent Skill 被确立为可保护的资产后，新的商业模式变得可能：

**能力授权即服务（Skill-as-a-Service）**。企业可以将自己的非核心但有价值的能力通过Level 2 授权方式对外提供，创造新的收入来源，同时保护核心竞争力不被泄露。

**能力联盟**。多家企业可以组建 Agent 协作联盟，在 ACN 框架下共同完成大型项目。每家企业贡献自己最强的能力，获得对应的收益分成，同时各自的核心 Skill 保持私有。这是一种比传统联合体更灵活、更安全的协作形式。

**能力估值与并购**。当企业 Skill 可以被量化评估（通过链上交付记录和声誉数据），企业并购中的「技术资产评估」将有了客观依据。这对中小企业尤其有利——它们的核心价值往往在于独特的能力，而不是固定资产。

## 五、技术实现的企业扩展

### 5.1 组织级 Skill 管理

个人的Skill管理相对简单——一个人决定自己的每个Skill的保护等级。但企业的Skill管理需要考虑组织层面的复杂性：

**Skill 注册与分类**。企业需要一个内部的 Skill 注册表，识别和分类所有通过 Agent系统积累的能力。哪些是核心竞争力（Level 1），哪些可以授权（Level 2），哪些可以开放（Level 3），需要有明确的治理流程。

**权限管理**。不同部门、不同项目的 Agent 可能有不同的 Skill 访问权限。企业需要在 ACN之上构建内部的权限管理层，确保 Skill 不会被内部的非授权使用所泄露。

**员工 Skill 与企业 Skill 的边界**。这是一个需要明确约定的灰色地带。员工在工作中培养的Skill，哪些属于个人，哪些属于企业？ASP框架建议：在入职时明确约定，员工为完成公司特定任务而培养的 Skill 归公司所有，但员工的通用专业能力和行业经验始终归属个人。这个约定应当透明、对等，而非单方面的格式条款。

### 5.2 企业间信任链

个人之间的信任可以通过声誉系统建立。企业之间的信任需要更丰富的维度：

ACN 的链上声誉系统在企业场景下可以扩展为：交付历史（完成质量和时效）、协议遵守记录（是否有违反 Skill保护条款的记录）、协作评分（合作方的主观评价）。这些数据共同构成一个企业的「Agent协作信用」，影响其在网络中获取优质合作机会的能力。

对于有过违反 Skill 保护条款记录的企业，网络可以自动降低其信用评级，限制其参与高价值协作的机会。这是一种去中心化的合规激励——不靠法律威慑，靠经济后果。

### 5.3 与现有企业系统的整合

ACN 不需要企业抛弃现有的 IT 基础设施。它可以作为一个协议层覆盖在企业现有的 Agent系统之上：

企业内部的Agent系统照常运行。当需要与外部Agent协作时，通过ACN网关进行交互——ACN 网关负责执行 Skill 保护策略、管理临时验证空间、处理智能合约和结算。这个架构类似于企业防火墙的逻辑：内部自由，外部受控。

## 六、行动呼吁

### 6.1 致企业决策者

如果你的企业正在部署Agent系统或者计划与外部Agent协作，你需要现在就开始思考技能保护策略。不是因为 ASP 已经是一个成熟的解决方案——它还不是。而是因为一旦你的核心能力在不受保护的协作中泄露，是不可逆的。

我们建议企业现在就着手：盘点自身的核心Agent Skill资产；评估当前跨企业Agent协作中的技能泄露风险；在采购和合作协议中加入 Agent Skill 保护条款；关注并参与 ASP/ACN的标准制定过程。

### 6.2 致行业协会和标准组织

Agent时代的产业协作规则不应该只由科技巨头来制定。行业协会有责任代表本行业企业的利益，在Agent 协作标准的制定中发出声音。ASP/ACN 提供了一个具体的讨论框架。

### 6.3 致 V1 的支持者

如果你已经认同个人 Agent 技能的主权主张，企业视角不是对个人主张的稀释，而是对它的增强。当企业也站在技能主权这一边时，这个运动的力量和影响力将完全不同。企业有预算、有法务、有产业影响力。它们是我们最需要的盟友。

但我们也要保持清醒：团结企业不是无条件的。ASP 的核心原则——包括保护个人技能产权——不能在争取企业支持的过程中被妥协。个人和企业在这个框架下是平等的参与者，遵守同样的规则，享有同样的权利。

> 「Agent 时代的真正分界线不是劳方与资方，不是大公司与小公司，而是创造者与剥夺者。ASP 站在所有创造者这一边。」

## 七、路线图展望
